We chose our dataset by brainstorming easily accessible information that is commonly misunderstood or misrepresented. This led us to choosing a dataset on government spending, specifically the US Federal Budget for the past 50 or so years. We chose to focus on the last 20 years since they hold the most relevance to us today.

To effectively display the data, we decided to go with an interactive stacked area chart that displays the amounts of spending per category (e.g., defense, welfare, and education) over time. We have chosen a stacked area chart because it allows the user to directly compare budget allocations for different categories. The user could get a better understanding on which categories receive more funding compared to others and how has that changed over years.

To make it easier for the user to understand which color in the graph corresponds to which category, we have included a legend on the top. Often times, users would like to explore a single or group of categories of interest more closely. To accommodate this, we have implemented category selection in the legend. The user can click on a particular category in the legend to check and uncheck it as well. Using this feature, the user can select any subset of categories to explore them more deeply. Giving the user the choice to explore categories more closely can help him/her gain a better understanding of the funding trend of individual categories and also allows him/her to more closely observe relationships among a subset of categories he/she might have encountered.

Additionally, if the user wants to explore the trend over a subset of years, they can do it by using the desired subset of years using the slider. By being able to explore the trend over specific years, the user can capture sudden hikes and drops and explore the reasons behind the corresponding hike or drop. For example, there was a sudden hike in the government spending of the “Other Spending” category in 2009. A likely reason for this would be the bank or auto bailouts of 2008, which don’t seem to fit into any other category. Basically, it allows the user to observe and find answers for any questions they might have for sudden or gradual changes in the trend.

Furthermore, we have added tooltips to our graph to make it easier for a user to compare and contrast data. Tooltips are visible upon hover and display the year and the levels of spending for this year. This allows the user to glean precise information from the chart. We also added a slider that allows the user to control the latest year to include in the visualization. We did this because the budgets past 2020 are not actual budgets, but are instead projected or estimated budgets.

We spent around 12-15 hours per person for this project. We did this by meeting up several times, for sessions from 2-5 hours long, as well as dividing up responsibilities to work on independently. This made our contributions roughly equal. Unfortunately, we spent a large majority of our time working on an implementation in D3, only to learn that Charts.js could much more easily implement the functions we hoped to include, for example animated transitions and multi-category filters. However, we’re glad we learned D3, since we plan on using it for our final project, due to its extreme flexibility.
